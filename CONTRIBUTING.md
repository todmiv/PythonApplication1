# Руководство для контрибьюторов

Спасибо за интерес к проекту! Ниже — минимальные правила и рекомендации, чтобы ваш вклад был принят быстро и без лишних итераций.

## Как начать
1. Форкните репозиторий и клонируйте свой форк.
2. Создайте ветку от `main` с осмысленным названием: `feature/<кратко>`, `fix/<кратко>`, `chore/<кратко>`.
3. Настройте виртуальное окружение и установите зависимости (см. README.md).

## Коммиты
- Пишите понятные сообщения: в повелительном наклонении, кратко и по делу.
- Примеры: `fix: исправить формирование заголовков таблицы`, `feat: добавить кэширование ответа API`.
- Один коммит — одна логическая единица изменений.

## Код‑стайл и качество
- Python 3.8+.
- По возможности используйте форматирование black и линтер ruff; типизацию mypy при расширении проекта.
- Пишите простые и понятные функции, избегайте дублирования.

## Тестирование
- В репозитории изначально нет тестов. Если добавляете новую логику — приветствуются тесты (pytest).

## Pull Request
1. Перед PR убедитесь, что проект запускается локально.
2. Описание PR должно содержать:
   - Что изменено и зачем.
   - Как проверять.
   - Возможные побочные эффекты.
3. Поддерживайте небольшой размер PR — легче ревьюить.

## Обсуждения и задачи
- Используйте Issues для багов и предложений.
- Приложите логи/скриншоты при необходимости и шаги для воспроизведения.

## Лицензия вклада
- Отправляя PR, вы соглашаетесь, что ваш вклад публикуется под лицензией репозитория (см. LICENSE).
